## Usage guidance

A `Link` is primarily used for navigation, and it usually appears _within_ or _directly following_ a sentence. For standalone navigation actions use `plain` [Button](/components/button) instead.

By default `Link` inherits its font size and weight from the surrounding text. On rare occasions, when a `Link` is used as standalone action, and not a part of a bigger text body, **always** set its `size`.

Third-party navigation components, such as [react-router](https://github.com/remix-run/react-router) `Link`, can be passed via the `as` prop.

## Best practices

<BestPractices>
  <BestPractices.Recommended>

    - open a new tab (with `target="blank"`) only when necessary
    - add clarity to the external Link through a clear label, rather than an icon
    - label should clearly describe Link purpose
    - use the same label for Links which navigate to the same content

  </BestPractices.Recommended>
  <BestPractices.Discouraged>

    - use for actions that don't appear within or directly following a sentence&mdash;use `plain` [Button](/components/button)
    - overuse icons, as they can add unnecessary visual clutter to the sentence
    - use Link for actions other than navigation&mdash;if a component that doesn't have URL is required, use [Button](/components/button)

  </BestPractices.Discouraged>
</BestPractices>
