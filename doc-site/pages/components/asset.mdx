## Usage guidance

Assets are **responsive** by default and will fit to their parent container if no `width` or `height` is provided (whilst still adhering to their `viewBox` aspect ratio).

They will also inherit the `color` of their parent container by default too.

For use-cases which are not React-based, we provide a download link for each Asset type below their examples, but otherwise the React components should be used.

## Best practices

Assets should be used to provide visual context or expand upon a written point. In most cases, they should **not** be used as a identifying marks and instead [Icons](/components/icon) should be used.

<BestPractices>
  <BestPractices.Recommended>

    - use [Icons](/components/icon) instead of certain Assets for small sizes

  </BestPractices.Recommended>
  <BestPractices.Discouraged>

    - use an Asset with too small a size, e.g. < 24px

  </BestPractices.Discouraged>
</BestPractices>

## Props

### Asset

Assets accept all props a regular `SVGElement` would, such as `width`, `height`, `aria-` and `data-` attributes.

## Imports

You can import any existing asset as a separate component. For example to import the `AvatarPlaceholderColored` asset use:

<Imports module={['AvatarPlaceholderColored']} path="@datacamp/waffles/asset" />
