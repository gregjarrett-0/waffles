## Usage guidance

Use `Avatars` to help identify a user, entity or group within a page, alongside additional information about what the `Avatar` represents. Visual-based `content` should be prioritized over text.

If an `Avatar` is used with an image of a user or individual, it should fallback to a placeholder image (for example the `AvatarPlaceholderColored` [Asset](http://localhost:3000/components/asset/#other-assets)) or the initial of the name when this image is unavailable. For smaller `size` variants, text-based `content` should always be used where an image is harder to identify.

Similarly, for smaller `Avatars` (below and including `small`) where an [Asset](/components/asset) is used for `content`, a relevant [Icon](/components/icon) should be used instead.

## Best practices

<BestPractices>
  <BestPractices.Recommended>

    - choose an appropriate content type for the specific Avatar size
    - prioritize visual-based `content`, where available
    - use `contentFill` when you have content that does not have a transparent background

  </BestPractices.Recommended>
  <BestPractices.Discouraged>

    - use images or assets for very small Avatars&mdash;use a single character or an [Icon](/components/icon) instead
    - use text for `content`, where a visual element is available&mdash;except for smaller sizes
    - use `contentFill` when using an [Icon](/components/icon) as content

  </BestPractices.Discouraged>
</BestPractices>
