## Usage guidance

Ideally, no more than one `Loader` should be visible on a single page at a time. For contexts where multiple areas have loading content (such as on page load), a form of skeleton loading should be used and the `Loader` as an overlay for the page/container.

The `Loader` will behave responsively to its' parent container, but a custom size can also be provided using the `height` or `width` property. Please note, both properties should not be provided together as this will break the responsive scaling behavior. The size of the `Loader` should also not fill the entire container.

For accessibility, `aria-label` should be provided for screen reader context and `aria-live`/`aria-busy` on the component that is dynamically modified as a result of the _loading_ task or that triggered the loading state.

## Best practices

<BestPractices>
  <BestPractices.Recommended>

    - Use when an action will take more than a few seconds to complete
    - Provide a simple but descriptive `aria-label` as context for screen readers&mdash;such as 'Loading user profile'
    - Use `aria-live` on the associated parent container, to allow for screen reader updates

  </BestPractices.Recommended>
  <BestPractices.Discouraged>

    - Use more than one `Loader` onscreen at once
    - Use to show progress&mdash;use [Progress](/components/progress) instead
    - Use as content within a [Button](/components/button)&mdash;use the `isLoading` Button property
    - Use when there is no content to show&mdash;use [Empty State](/components/empty-state)

  </BestPractices.Discouraged>
</BestPractices>
