## Usage guidance

Use `ErrorBoundary` to handle JavaScript runtime errors. Try to at least wrap a few major sections of the app with it, although the more localized it is the better.

When logging errors to 3rd party services, such as Sentry, use the `onError` callback.

## Best practices

<BestPractices>
  <BestPractices.Recommended>

    - usually use default fallback to keep things consistent
    - wrap many sections of the app
    - for custom error messages provided via `fallback`, try to incorporate recovery strategy with `onReset` callback

  </BestPractices.Recommended>
  <BestPractices.Discouraged>

    - use to handle regular data fetch requests errors
    - use only in the root of the app

  </BestPractices.Discouraged>
</BestPractices>
